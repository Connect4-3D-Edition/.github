# .github Repo

## Purpose

This repo holds shared GitHub organization/profile content, not application code.

When this repo describes the project publicly, it should reflect the canonical product framing:

- product name: `Connect4 | 3D-Edition`
- handle: `Connect4-3D-Edition`
- abbreviation: `c4-3d`
- genre framing: a serious competitive online strategy game, not just a casual board-game clone
- core concept: a `4 x 4 x 4` Connect Four variant played on 16 poles with gravity per pole and four-in-a-row win detection in 3D space

Public-facing copy should emphasize strategic depth, fairness, competitive clarity, and replayability. Avoid overselling speculative implementation details.

## Main location

- `profile/README.md`: public organization profile page

When the developer cockpit changes materially, the profile README should point developers to the `.cockpit` repository as the starting point for workspace setup and local onboarding.

## Working rule for the agent

Use this repo for GitHub-facing metadata only. If the task is about product code, switch to one of the service, frontend, shared-library, migration, or infrastructure repos.
